# Contribuer au guide

Merci de vouloir améliorer ce manuel.

## Avant de proposer une modification

Une leçon doit rester compréhensible par une personne qui découvre le sujet. Elle peut utiliser du vocabulaire technique, mais ce vocabulaire doit être expliqué.

Pour une nouvelle fonction GML :

1. vérifier la syntaxe dans le manuel officiel actuel ;
2. tester l'exemple dans GameMaker ;
3. préciser dans quel événement le code est utilisé ;
4. expliquer chaque variable importante ;
5. ajouter au moins un cas limite ou une erreur fréquente ;
6. mettre un lien vers la page officielle correspondante.

## Style d'écriture

- écrire en français simple ;
- parler directement au lecteur quand cela aide ;
- préférer un petit exemple vérifiable à un gros bloc magique ;
- ne pas présenter un réglage de game design comme une vérité universelle ;
- expliquer pourquoi le code existe, pas seulement où le coller ;
- éviter de recopier le manuel officiel ;
- signaler clairement les approximations et les choix pédagogiques.

## Exemples et ressources

Les exemples doivent être génériques et ne pas dépendre d'assets payants ou dont la licence est incertaine.

Les captures, schémas et illustrations ajoutés au dépôt doivent être originaux ou compatibles avec la licence du projet. Leur origine doit être documentée si nécessaire.

## Vérification minimale

Avant une proposition :

- ouvrir `index.html` dans un navigateur ;
- vérifier le sommaire et les liens internes ;
- vérifier l'affichage sur une fenêtre étroite ;
- vérifier les liens vers le manuel GameMaker ;
- relire les exemples GML dans l'ordre où un débutant les découvrira.
